## 1. 앱 개요

**SKINDESK**는 피부·미용 시술을 받는 개인을 위한 **개인 피부 코디네이터 앱**이다.

병원 시술 기록, 시술권 잔여 현황, 포인트/결제 내역, 시술 주기 관리, 예약 일정을 한 곳에서 관리한다.

### 핵심 타겟
- 피부과·미용 클리닉을 **정기적으로 방문**하는 20~40대 여성
- 2개 이상의 병원을 이용하며, 시술 기록이 분산되어 있는 사용자
- 고단가 시술(리프팅, 보톡스, 필러 등)을 주기적으로 관리하는 사용자

### 앱 슬로건
*"Your Personal Skin Coordinator"*

## 2. 핵심 기능 (v1.0 기준)

### 2-1. 시술 기록 관리
- 병원별 시술 날짜, 시술명, 금액, 만족도, 메모 기록
- 시술 레이어 분류: 표피층(토닝·필링) / 진피층(보톡스·리쥬란·필러) / 피하조직(리프팅·타이트닝)
- 샷수별 세분화 기록 (슈링크 100/200/300/600샷, 써마지 300/600샷 등)

### 2-2. 시술 주기 관리
- 71종 시술 × 샷수별 권장 주기 데이터 보유
- 관리 강도 설정: 💪 타이트 / 🔵 유지 / ⚫ 안함 (부위별)
- D-7 / D-0 알림 (예정)
- 캘린더 연동 (예정)

### 2-3. 남은 시술 & 포인트
- 병원별 잔여 시술권 현황 (몇 회 남았는지)
- 병원별 포인트 잔액
- 밴스 계열: 포인트 충전 = 실제 결제 구조 반영
- 타의원: 시술 직접 결제 구조 반영

### 2-4. 결제 내역
- 병원 유형별(밴스/타의원) 결제 분리
- 누적 지출 통계, 병원별 지출 비중 시각화

### 2-5. 병원 등록
- 카카오 로컬 API 기반 병원 검색 및 자동완성 (개발 예정)
- 사용자가 직접 병원 추가 가능

### 2-6. 다국어 지원
- 한국어 / English / 简体中文
- 로그인 화면부터 언어 선택 가능

## 3. 기술 스택 및 데이터 구조

### 프론트엔드
- Vite + React + TypeScript + Tailwind CSS + shadcn/ui
- 배포: Lovable (skindesk.lovable.app)
- 모바일 앱 변환: Capacitor (Android 우선)

### 백엔드 / DB
- **Supabase** (PostgreSQL)
- 인증: Supabase Auth (이메일 + Google OAuth + 카카오 OAuth)
- 보안: RLS(Row Level Security) — 유저별 데이터 완전 분리

### 핵심 DB 테이블
```
treatment_records     시술 기록 (날짜, 시술명, 금액, 만족도, 메모)
treatment_packages    시술권 (잔여 횟수, 만료일)
point_transactions    포인트 내역 (충전/사용/잔액)
payment_records       결제 내역 (병원 유형별)
treatment_cycles      시술 주기 설정
user_profiles         사용자 정보 (생년월일, 피부타입, 관심부위)
clinic_balances       병원별 잔액
```

### 수집 데이터 (사용자 입력 기반)
- 병원명, 시술 종류, 시술 금액, 결제 방식
- 시술 날짜, 만족도 (1~5점), 메모/후기
- 생년월일, 피부 타입, 주요 피부 고민, 관리 목표
- 잔여 시술권, 포인트 잔액, 결제 이력
- 시술 주기 설정값, 알림 설정

## 4. 경쟁 앱 분석

### 현재 존재하는 앱들
| 앱 유형 | 예시 | 한계 |
|---------|------|------|
| 병원 예약 앱 | 강남언니, 굿닥, 힐링페이퍼 | 병원 마케팅 플랫폼. 개인 기록 관리 없음 |
| 가계부 앱 | 뱅크샐러드, 토스 | 피부 시술 특화 없음. 잔여 시술권 개념 없음 |
| 메모/다이어리 앱 | 노션, 네이버 메모 | 주기 계산, 알림, 시술 분류 없음 |
| 건강 기록 앱 | 삼성헬스 등 | 미용 시술 카테고리 없음 |

### 결론
**"피부 시술"에 특화된 개인 기록 관리 앱은 현재 시장에 없다.**

## 5. 타 앱이 따라오기 어려운 차별화 요소

### 5-1. 도메인 특화 데이터 구조
일반 앱은 피부 시술의 복잡한 구조를 모른다.

- 시술 레이어(표피/진피/피하조직) 분류 체계
- 샷수별 권장 주기 차등화 (슈링크 300샷 vs 600샷의 회복 주기가 다름)
- 병원마다 다른 포인트 충전 구조 (밴스처럼 선불 포인트제 운영하는 병원)
- 시술권 잔여 관리 (10회권 중 4회 남음)

이 구조를 처음부터 설계하지 않으면 나중에 데이터가 쌓여도 의미 없는 데이터가 된다.

### 5-2. 실사용자 데이터로 검증된 71종 시술 주기 데이터
- 슈링크/울쎄라/세르프/써마지/보톡스/리쥬란 등 71종
- 샷수별, 관리 강도별(타이트/유지) 세분화
- 딥 리서치 + 실사용 데이터로 보정
- 경쟁 서비스가 만들려면 의료 자문 + 수년간 데이터 필요

### 5-3. 멀티 클리닉 통합 관리
강남언니·굿닥은 플랫폼 내 병원만 관리한다.
SKINDESK는 **어느 병원이든** 기록할 수 있다.
- 밴스(선불 포인트제) + 필로의원(일반 결제) + 뷰티라운지(세션권) 동시 관리
- 사용자 입장에서 "내 모든 피부 관리 이력"이 한 곳에

### 5-4. 사용자가 직접 쌓는 데이터 = 전환 비용(Lock-in)
사용자가 2년치 시술 기록을 SKINDESK에 쌓으면 다른 앱으로 이동하지 않는다.
이것이 가장 강력한 리텐션 메커니즘이다.

- 기록이 쌓일수록 주기 추천 정확도 상승
- 개인 맞춤 인사이트 증가 ("작년 이맘때 이 시술 받았는데 만족도 5점이었어요")
- 지출 이력 → 연간 피부 관리 비용 파악

### 5-5. 익명·통계 데이터 자산
사용자가 늘수록 아무도 갖지 못한 데이터가 쌓인다.

```
예시 인사이트:
"세르프 600샷 평균 만족도 4.1 / 재시술 주기 중앙값 172일"
"30대 초반 여성, 밴스 계열 평균 연 지출 320만원"
"리쥬란힐러 단독 vs 콤보 시술 만족도 비교"
"특정 클리닉의 특정 시술 만족도 실사용자 집계"
```

이 데이터는 향후:
- 병원 B2B 서비스 (우리 고객 리텐션 데이터 제공)
- 시술 추천 알고리즘 정교화
- 데이터 바우처 사업 연계 (의료 데이터 판매)
- 광고 플랫폼 (타겟 광고 높은 정확도)

으로 연결 가능하다.
